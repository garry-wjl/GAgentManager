# GAgentManager 项目文档命名规范

## 1. 总体命名原则

1. **一致性**：所有文档遵循统一的命名规范
2. **可读性**：文件名应当易于理解和识别
3. **时效性**：包含版本号或日期信息以便追踪更新
4. **功能性**：通过文件名可以明确文档用途

## 2. 各目录文件命名规则

### 2.1. PRD文档 (prd/)

**命名格式：** `PRD-[功能模块]-[版本号].md`

**示例：**
- `PRD-Agent管理-V1.0.md`
- `PRD-用户管理-V1.2.md`
- `PRD-模型管理-V2.0.md`
- `PRD-Skill商店-V1.5.md`
- `PRD-MCP管理-V1.1.md`
- `PRD-工作流管理-V1.3.md`

**版本号规则：**
- 主版本号.次版本号（如 V1.0, V1.1, V2.0）
- 初始版本为 V1.0
- 重要更新升级主版本号
- 小幅修改升级次版本号

### 2.2. 技术方案文档 (技术方案/)

**命名格式：** `TS-[技术领域]-[具体功能]-[版本号].md`

**示例：**
- `TS-Backend-API设计-V1.0.md`
- `TS-Frontend-架构设计-V1.2.md`
- `TS-Database-数据库设计-V1.1.md`
- `TS-Deployment-部署方案-V1.0.md`
- `TS-Security-安全方案-V1.3.md`
- `TS-Performance-性能优化-V1.0.md`

**技术领域缩写：**
- Backend (后端)
- Frontend (前端)
- Database (数据库)
- Deployment (部署)
- Security (安全)
- Performance (性能)
- Integration (集成)

### 2.3. 测试方案文档 (测试方案/)

**命名格式：** `TT-[测试类型]-[测试对象]-[版本号].md`

**示例：**
- `TT-Functional-Agent管理功能-V1.0.md`
- `TT-Performance-性能测试-V1.2.md`
- `TT-Security-安全测试-V1.1.md`
- `TT-Integration-集成测试-V1.0.md`
- `TT-UAT-用户验收测试-V1.0.md`
- `TT-Automation-自动化测试-V1.1.md`

**测试类型缩写：**
- Functional (功能测试)
- Performance (性能测试)
- Security (安全测试)
- Integration (集成测试)
- UAT (用户验收测试)
- Unit (单元测试)
- Automation (自动化测试)

### 2.4. 规划文档 (issue/)

**命名格式：** `PLAN-[规划类型]-[主题]-[日期].md`

**示例：**
- `PLAN-Roadmap-产品路线图-2026Q2.md`
- `PLAN-Milestone-Q2里程碑-202604.md`
- `PLAN-Tasks-开发任务-20260428.md`
- `PLAN-Issue-Issue跟踪-202604.md`
- `PLAN-Sprint-Sprint计划-2026W17.md`

**规划类型缩写：**
- Roadmap (产品路线图)
- Milestone (里程碑)
- Tasks (任务列表)
- Issue (Issue跟踪)
- Sprint (Sprint计划)
- Budget (预算规划)
- Resource (资源规划)

### 2.5. 其他文档 (other/)

**命名格式：** `[类型]-[主题]-[可选版本].md`

**示例：**
- `Note-会议纪要-20260428.md`
- `Draft-概念草案-V0.1.md`
- `Template-文档模板.md`
- `Guide-使用指南.md`
- `FAQ-常见问题.md`
- `Glossary-术语表.md`

## 3. 通用规则

### 3.1. 日期格式
- YYYYMMDD (如 20260428 表示 2026年4月28日)
- YYYYQx (如 2026Q2 表示 2026年第二季度)
- YYYYWxx (如 2026W17 表示 2026年第17周)

### 3.2. 版本号格式
- V主版本号.次版本号 (如 V1.0, V2.1)
- 初始版本 V1.0
- 重要变更升级主版本号
- 修正更新升级次版本号

### 3.3. 特殊字符规则
- 使用连字符 `-` 分隔各部分
- 不使用空格、特殊符号
- 英文使用半角字符
- 文件名长度不超过50个字符

### 3.4. 文件后缀规范
- `.md` - Markdown格式文档
- `.xlsx` - Excel电子表格
- `.drawio` - 流程图文件
- `.pdf` - PDF格式文档

## 5. 文件命名检查清单

在创建新文档时，请确保：
- [ ] 使用规定的命名格式
- [ ] 包含正确的缩写标识
- [ ] 版本号正确
- [ ] 日期格式规范
- [ ] 文件类型正确
- [ ] 长度适中
- [ ] 无特殊字符
- [ ] 易于理解

## 6. 规范目的

遵循此命名规范有助于：
1. 快速识别文档类型和内容
2. 有效管理文档版本
3. 方便文档检索和归档
4. 维护项目文档的规范性
