# GAgentManager 测试方案

## 1. 测试范围

| 系统 | 模块 | 优先级 |
|------|------|--------|
| 用户端 | 登录/注册、Agent对话、个人中心 | P0 |
| 管理端 | Agent管理、用户管理、权限管理、模型管理、MCP管理、Skill商店、系统配置 | P0 |
| 公共 | 认证鉴权、审计日志、通知中心 | P1 |
| 性能/安全 | 接口性能、安全漏洞 | P2 |

## 2. 测试策略

### 2.1 单元测试
- **目标**: 代码行覆盖率 >= 80%，核心业务逻辑 100%
- **工具**: JUnit 5 + Mockito（后端），Vitest（前端）
- **覆盖**: Service 层业务逻辑、工具类、数据转换器

### 2.2 集成测试
- **目标**: 验证各模块间接口调用正确性
- **工具**: Spring Boot Test + H2 内存库、MockMvc
- **覆盖**: API 接口、数据库操作、Redis 缓存、JWT 认证

### 2.3 E2E 测试
- **目标**: 关键用户流程端到端验证
- **工具**: Playwright
- **覆盖**: 登录→对话、Agent创建→发布→上线、权限配置

### 2.4 手动测试
- 兼容性测试（Chrome/Firefox/Safari/Edge）
- 响应式布局验证
- 用户体验走查

## 5. 安全测试

| 场景 | 检查点 |
|------|--------|
| SQL注入 | 所有输入参数做SQL注入测试 |
| XSS攻击 | 消息内容、Agent名称等富文本字段做XSS注入 |
| CSRF | 管理端接口需验证CSRF Token |
| 越权访问 | 水平越权（访问他人数据）/垂直越权（越级操作） |
| 敏感数据 | 密码、Token不在日志中明文显示 |
| 暴力破解 | 登录失败锁定、验证码机制生效 |
| 文件上传 | 限制文件类型、大小，防止恶意文件 |
