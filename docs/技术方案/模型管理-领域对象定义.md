# 模型管理 - 领域对象属性与界面表单定义

> 文档版本：V1.0
> 创建日期：2026-04-28
> 关联 PRD：[PRD-GAgentManager-V5.0](../prd/PRD-GAgentManager-V5.0.md) §4.1.7

## 核心领域对象

模型管理包含以下核心领域对象：

| 领域对象 | 说明 |
|---------|------|
| **Model（模型）** | 大语言模型的核心实体，包含基本信息、连接配置、参数配置、能力标签、状态等 |
| **ModelQuota（模型配额）** | 模型的使用配额配置，包含每日 Token 上限、请求次数上限等 |
| **ModelRateLimit（模型限流）** | 模型的调用频率限制，包含 QPS/TPM 限制等 |
| **ModelCost（模型成本）** | 模型的计费配置，包含输入/输出 Token 单价等 |
| **AgentModelBinding（Agent-模型绑定）** | Agent 与模型之间的关联关系 |

> **设计说明**：Quota、RateLimit、Cost 在数据库层面可作为 Model 表的字段组（内联），不需要独立建表。在代码设计中可作为值对象（Value Object）封装。

## 1. Model（模型）

### 1.1 属性定义

#### 1.1.1 基本信息

| 字段名 | 字段代码 | 类型 | 必填 | 约束/校验 | 默认值 | 说明 |
|-------|---------|------|------|----------|--------|------|
| ID | `id` | Long | 系统生成 | 主键，自增 | - | 唯一标识 |
| 模型编码 | `modelCode` | String(64) | 是 | 全局唯一，小写字母+数字+下划线，3-64字符 | - | 模型内部标识，如 `gpt-4o`、`deepseek-chat` |
| 模型名称 | `modelName` | String(128) | 是 | 1-128字符，可重复 | - | 展示名称，如 `GPT-4o`、`DeepSeek Chat` |
| 提供商 | `provider` | Enum | 是 | OPENAI / DEEPSEEK / ANTHROPIC / DASHSCOPE / WENXIN / ZHIPU / OLLAMA / CUSTOM | - | 模型提供商 |
| API 类型 | `apiType` | Enum | 是 | OPENAI_COMPATIBLE / ANTHROPIC / DASHSCOPE / WENXIN / CUSTOM | - | API 协议类型，决定调用适配方式 |
| 描述 | `description` | String(512) | 否 | 最大 512 字符 | - | 模型用途说明 |
| 分类/标签 | `category` | String(64) | 否 | 预设值或自定义 | GENERAL | 如 GENERAL、CODING、VISION、EMBEDDING |

#### 1.1.2 连接配置

| 字段名 | 字段代码 | 类型 | 必填 | 约束/校验 | 默认值 | 说明 |
|-------|---------|------|------|----------|--------|------|
| API Key | `apiKey` | String(512) | 是 | 加密存储，最大 512 字符 | - | 访问模型的凭证 |
| Base URL | `baseUrl` | String(256) | 是 | 合法 URL 格式 | - | API 端点地址，支持自定义（如 `https://api.openai.com/v1`） |
| 超时时间（秒） | `timeout` | Integer | 是 | 1 ~ 600 | 60 | 单次请求最大等待时间 |
| 重试次数 | `retryCount` | Integer | 是 | 0 ~ 5 | 3 | 请求失败时的重试次数 |
| 重试间隔（毫秒） | `retryInterval` | Integer | 是 | 100 ~ 10000 | 1000 | 重试之间的间隔时间 |

#### 1.1.3 参数配置

| 字段名 | 字段代码 | 类型 | 必填 | 约束/校验 | 默认值 | 说明 |
|-------|---------|------|------|----------|--------|------|
| 最大 Token 数 | `maxTokens` | Integer | 是 | 1 ~ 模型实际最大上限 | 4096 | 单次响应最大 Token 数 |
| 温度默认值 | `temperature` | Double | 是 | 0.0 ~ 2.0 | 0.7 | 控制输出随机性 |
| 温度最小值 | `temperatureMin` | Double | 否 | 0.0 ~ temperature | 0.0 | Agent 可调整的下限 |
| 温度最大值 | `temperatureMax` | Double | 否 | temperature ~ 2.0 | 1.0 | Agent 可调整的上限 |
| Top-P 默认值 | `topP` | Double | 否 | 0.0 ~ 1.0 | 1.0 | 核采样参数 |
| Top-K 默认值 | `topK` | Integer | 否 | 1 ~ 100 | -1（不限制） | Top-K 采样参数 |
| 频率惩罚 | `frequencyPenalty` | Double | 否 | -2.0 ~ 2.0 | 0.0 | 降低重复内容的权重 |
| 存在惩罚 | `presencePenalty` | Double | 否 | -2.0 ~ 2.0 | 0.0 | 鼓励新话题的权重 |

#### 1.1.4 能力标签

| 字段名 | 字段代码 | 类型 | 必填 | 约束/校验 | 默认值 | 说明 |
|-------|---------|------|------|----------|--------|------|
| 能力标签 | `capabilities` | JSON Array | 是 | 非空数组 | - | 模型支持的能力列表 |

能力标签枚举值：

| 能力值 | 说明 |
|-------|------|
| `CHAT` | 对话/补全（基础能力，所有模型必须具备） |
| `FUNCTION_CALL` | 函数调用 / 工具调用 |
| `MULTI_MODAL` | 多模态输入（图片/音频/视频） |
| `JSON_MODE` | JSON 输出模式 |
| `STRUCTURED_OUTPUT` | 结构化输出（严格 JSON Schema） |
| `STREAMING` | 流式输出（SSE） |
| `EMBEDDING` | 文本向量化 |

#### 1.1.5 状态与审计

| 字段名 | 字段代码 | 类型 | 必填 | 约束/校验 | 默认值 | 说明 |
|-------|---------|------|------|----------|--------|------|
| 启用状态 | `status` | Enum | 是 | ENABLED / DISABLED | ENABLED | 模型是否对平台可用 |
| 连通性状态 | `connectivityStatus` | Enum | 系统更新 | OK / ERROR / UNTESTED | UNTESTED | 最近一次连通性测试结果 |
| 连通性测试时间 | `lastTestTime` | DateTime | 系统记录 | - | - | 最近一次连通性测试时间 |
| 创建人 | `createdBy` | String(64) | 系统记录 | - | - | 操作人 |
| 创建时间 | `createTime` | DateTime | 系统记录 | - | 当前时间 | 创建时间 |
| 更新人 | `updatedBy` | String(64) | 系统记录 | - | - | 最后修改人 |
| 更新时间 | `updateTime` | DateTime | 系统记录 | - | 当前时间 | 最后修改时间 |
| 逻辑删除 | `deleted` | Boolean | 系统记录 | 0 / 1 | 0 | 软删除标记 |

#### 1.1.6 成本配置（值对象，内联到 Model 表）

| 字段名 | 字段代码 | 类型 | 必填 | 约束/校验 | 默认值 | 说明 |
|-------|---------|------|------|----------|--------|------|
| 计费模式 | `cost.billingMode` | Enum | 是 | TOKEN_BASED | TOKEN_BASED | 当前仅支持按 Token 计费 |
| 输入 Token 单价 | `cost.inputPricePer1K` | BigDecimal | 是 | >= 0，最多 6 位小数 | 0 | 每 1000 输入 Token 的价格（元） |
| 输出 Token 单价 | `cost.outputPricePer1K` | BigDecimal | 是 | >= 0，最多 6 位小数 | 0 | 每 1000 输出 Token 的价格（元） |
| 货币单位 | `cost.currency` | String(8) | 是 | CNY / USD | CNY | 价格货币 |

#### 1.1.7 配额管理（值对象，内联到 Model 表）

| 字段名 | 字段代码 | 类型 | 必填 | 约束/校验 | 默认值 | 说明 |
|-------|---------|------|------|----------|--------|------|
| 每日最大 Token 数 | `quota.dailyMaxTokens` | Long | 否 | > 0，0 表示不限制 | 0 | 每日该模型最大 Token 消耗量 |
| 每日最大请求数 | `quota.dailyMaxRequests` | Long | 否 | > 0，0 表示不限制 | 0 | 每日该模型最大请求次数 |
| 每月最大 Token 数 | `quota.monthlyMaxTokens` | Long | 否 | > 0，0 表示不限制 | 0 | 每月该模型最大 Token 消耗量 |
| 每月最大请求数 | `quota.monthlyMaxRequests` | Long | 否 | > 0，0 表示不限制 | 0 | 每月该模型最大请求次数 |

#### 1.1.8 速率限制（值对象，内联到 Model 表）

| 字段名 | 字段代码 | 类型 | 必填 | 约束/校验 | 默认值 | 说明 |
|-------|---------|------|------|----------|--------|------|
| QPS 限制 | `rateLimit.maxQPS` | Integer | 否 | > 0，0 表示不限制 | 0 | 每秒最大请求数 |
| TPM 限制 | `rateLimit.maxTPM` | Long | 否 | > 0，0 表示不限制 | 0 | 每分钟最大 Token 数（Tokens Per Minute） |
| 并发限制 | `rateLimit.maxConcurrent` | Integer | 否 | > 0，0 表示不限制 | 0 | 最大并发请求数 |

## 2. AgentModelBinding（Agent-模型绑定）

此领域对象描述 Agent 与已注册模型之间的关联关系。模型管理页面仅展示关联信息，绑定关系的核心操作在 Agent 管理页面。

### 2.1 属性定义

| 字段名 | 字段代码 | 类型 | 必填 | 约束/校验 | 默认值 | 说明 |
|-------|---------|------|------|----------|--------|------|
| ID | `id` | Long | 系统生成 | 主键，自增 | - | 唯一标识 |
| Agent ID | `agentId` | Long | 是 | 外键，关联 Agent 表 | - | 所属 Agent |
| 模型 ID | `modelId` | Long | 是 | 外键，关联 Model 表 | - | 绑定的模型 |
| 是否默认 | `isDefault` | Boolean | 是 | - | false | 是否为该 Agent 的默认模型 |
| 排序序号 | `sortOrder` | Integer | 是 | >= 0 | 0 | 在 Agent 模型列表中的排序 |
| 创建人 | `createdBy` | String(64) | 系统记录 | - | - | 操作人 |
| 创建时间 | `createTime` | DateTime | 系统记录 | - | 当前时间 | 创建时间 |

### 2.2 界面展示（模型详情页"已关联 Agent"区域）

| 列名 | 字段 | 组件 | 说明 |
|-----|------|------|------|
| Agent 名称 | `agentId` → Agent 名称 | 链接 | 点击跳转到 Agent 详情 |
| Agent 编码 | `agentId` → Agent 编码 | 文本 | - |
| 是否默认 | `isDefault` | 状态标签 | 是/否 |
| 绑定时间 | `createTime` | 文本 | - |

## 3. 数据库表设计（MySQL）

### 3.1 model 表

```sql
CREATE TABLE `model` (
    `id`                  BIGINT          NOT NULL AUTO_INCREMENT COMMENT '主键',
    `model_code`          VARCHAR(64)     NOT NULL                COMMENT '模型编码',
    `model_name`          VARCHAR(128)    NOT NULL                COMMENT '模型名称',
    `provider`            VARCHAR(32)     NOT NULL                COMMENT '提供商',
    `api_type`            VARCHAR(32)     NOT NULL                COMMENT 'API类型',
    `description`         VARCHAR(512)    DEFAULT NULL            COMMENT '描述',
    `category`            VARCHAR(64)     DEFAULT 'GENERAL'       COMMENT '分类',

    `api_key`             VARCHAR(512)    NOT NULL                COMMENT 'API Key（加密存储）',
    `base_url`            VARCHAR(256)    NOT NULL                COMMENT 'API端点地址',
    `timeout`             INT             NOT NULL DEFAULT 60     COMMENT '超时时间（秒）',
    `retry_count`         INT             NOT NULL DEFAULT 3      COMMENT '重试次数',
    `retry_interval`      INT             NOT NULL DEFAULT 1000   COMMENT '重试间隔（毫秒）',

    `max_tokens`          INT             NOT NULL DEFAULT 4096   COMMENT '最大Token数',
    `temperature`         DECIMAL(3,2)    NOT NULL DEFAULT 0.70   COMMENT '温度默认值',
    `temperature_min`     DECIMAL(3,2)    DEFAULT 0.00            COMMENT '温度最小值',
    `temperature_max`     DECIMAL(3,2)    DEFAULT 1.00            COMMENT '温度最大值',
    `top_p`               DECIMAL(3,2)    DEFAULT 1.00            COMMENT 'Top-P',
    `top_k`               INT             DEFAULT -1              COMMENT 'Top-K',
    `frequency_penalty`   DECIMAL(3,2)    DEFAULT 0.00            COMMENT '频率惩罚',
    `presence_penalty`    DECIMAL(3,2)    DEFAULT 0.00            COMMENT '存在惩罚',

    `capabilities`        JSON            NOT NULL                COMMENT '能力标签',

    `status`              VARCHAR(16)     NOT NULL DEFAULT 'ENABLED' COMMENT '启用状态',
    `connectivity_status` VARCHAR(16)     NOT NULL DEFAULT 'UNTESTED' COMMENT '连通性状态',
    `last_test_time`      DATETIME        DEFAULT NULL            COMMENT '最后测试时间',
    `test_error_message`  VARCHAR(1024)   DEFAULT NULL            COMMENT '测试错误信息',

    `billing_mode`        VARCHAR(16)     NOT NULL DEFAULT 'TOKEN_BASED' COMMENT '计费模式',
    `input_price_per_1k`  DECIMAL(10,6)   NOT NULL DEFAULT 0      COMMENT '输入Token单价（元/千）',
    `output_price_per_1k` DECIMAL(10,6)   NOT NULL DEFAULT 0      COMMENT '输出Token单价（元/千）',
    `currency`            VARCHAR(8)      NOT NULL DEFAULT 'CNY'  COMMENT '货币单位',

    `daily_max_tokens`    BIGINT          DEFAULT 0               COMMENT '每日最大Token数',
    `daily_max_requests`  BIGINT          DEFAULT 0               COMMENT '每日最大请求数',
    `monthly_max_tokens`  BIGINT          DEFAULT 0               COMMENT '每月最大Token数',
    `monthly_max_requests`BIGINT          DEFAULT 0               COMMENT '每月最大请求数',
```
