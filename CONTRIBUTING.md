# 贡献指南

感谢您有兴趣为 GAgentManager 项目做出贡献！本文档提供了有关如何参与项目开发的相关信息。

## 开发环境搭建

1. 克隆项目到本地
2. 根据不同模块的README文档安装依赖和运行项目
3. 通常情况下，每个模块都有自己的依赖和启动方式

## 代码风格

- 遵循各语言/框架的最佳实践
- 保持代码的一致性和可读性
- 提交前请确保代码经过适当的测试

## 提交规范

- 提交信息应简洁明了地描述更改内容
- 对于重大变更，请先提交 Issue 进行讨论
- Pull Request 应包含详细的变更说明

## 分支策略

- 主要开发分支为 `main` 
- 为新功能创建独立的功能分支
- 修复 bug 时创建 hotfix 分支

## 问题报告

- 在提交问题之前，请搜索是否已有类似问题
- 详细描述问题现象、复现步骤和预期行为
- 如有可能，请提供相关日志或截图

## 代码审查

- 所有代码都需要经过至少一名维护者的审查
- 保持积极的沟通态度，建设性地提出改进建议
- 尊重其他贡献者的时间和努力

我们欢迎任何形式的贡献，包括但不限于代码、文档、测试、错误报告等。让我们一起让 GAgentManager 变得更好！
